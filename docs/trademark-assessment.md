# Trademark / Name-Availability Assessment — "Scribed" (macOS app)

> **DECISION (2026-06-26):** Acting on this assessment, the product was **renamed from
> "Scribed" to "Seshat"** (Egyptian goddess of writing/records). This document is kept as the
> record of *why*. A formal clearance search on "Seshat" is still recommended before store
> submission; initial screening found no Mac transcription/note app using the name.

**Subject:** A macOS menu-bar app named **Scribed** that turns meeting recordings into
Markdown notes via the user's own WhisperX + Ollama servers (local-first, bring-your-own-server).

**Date of assessment:** 2026-06-25
**Author:** Joan Marc Riera Duocastella (with research assistance)
**Status of this document:** Informal, evidence-based screening — **not legal advice** and **not a
formal clearance search.** Public register searches below were partly inconclusive; see the
"Before submission" checklist for the formal steps.

---

## TL;DR verdict

**Risk of shipping a Mac app called "Scribed" in the transcription / meeting-notes niche:
MEDIUM–HIGH.**

There is a **live, directly-competing commercial product** already trading as **"Scribed AI"**
(scribed.ai, operated by Q6 Technology Holdings, Inc.) whose feature set explicitly includes
**AI meeting transcription** — the same space this app sits in. The name "Scribed" is also a
near-homophone of the registered mark **SCRIBD** (Scribd, Inc.) and sits inside a very crowded
"Scribe / Scribe-*" cluster of transcription tools. Every reasonable "Scribed" domain (.com, .io,
.ai, getscribed.com) is already taken. **Recommendation: rename.** Candidate names and a "before
submission" checklist are below.

---

## 1. Trademark registers

Goal: live **word-marks** on "Scribed" in software-relevant Nice classes — **Class 9** (software,
downloadable apps) and **Class 42** (SaaS / software services).

> Caveat: the live USPTO / EUIPO / UK IPO register front-ends are JavaScript search apps that do
> not return record data to automated fetching, so the searches below relied on third-party
> mirrors (uspto.report, Justia/Trademarkia) plus general web search. **Treat the "no exact
> 'Scribed' mark found" results as inconclusive, not as a clean bill of health.** A paid/formal
> clearance search is required before relying on this.

### USPTO (United States) — tmsearch.uspto.gov

- **No exact live "Scribed" word-mark** surfaced via public mirrors for Class 9 / 42. This is
  **inconclusive** (the official register was not directly queryable here).
- **Highly relevant nearby live marks** that an examiner could cite for likelihood of confusion:
  - **SCRIBD** — Scribd, Inc., Reg. No. 3777227, Serial 77795968. Covers software for sharing
    electronic documents over networks. **"Scribed" is phonetically identical to "Scribd."**
    Source: <https://trademarks.justia.com/777/95/scribd-77795968.html>
  - **SCRIBEMATIC** — Autoscribe, L.L.C., Serial 98050156. AI software for recording/notes
    (clinical). Source: <https://uspto.report/TM/98050156>
  - **SCRIBEBRIDGE** — TapRoot Systems, Inc., Serial 88853923. SaaS for medical scribing.
    Source: <https://uspto.report/TM/88853923>
  - **SCRIB3D** — SCRIB3D LLC, Reg. 6521867. Source:
    <https://trademarks.justia.com/900/42/scrib3d-90042458.html>
- Takeaway: the **"SCRIB-/SCRIBE-" root is heavily occupied** in software classes, and a
  phonetically identical registered mark (SCRIBD) exists for document/content software. Even
  absent an exact "Scribed" registration, **adopting "Scribed" invites a refusal or opposition
  on likelihood-of-confusion grounds** if you ever try to register it.

### EUIPO (European Union) — euipo.europa.eu (eSearch plus)

- **Public search inconclusive** — the EUIPO eSearch front-end did not return record data to
  automated queries. No exact live "Scribed" EUTM was confirmed or ruled out.
- General-purpose searches surfaced no obvious "Scribed" EUTM, but this **must be verified** via
  eSearch plus / TMview before any reliance. Source: <https://www.euipo.europa.eu/en/trade-marks>
  and TMview <https://www.tmdn.org/tmview/>.

### UK IPO (United Kingdom) — gov.uk trade-mark search

- **Public search inconclusive** — same limitation. No exact live "Scribed" UK mark was confirmed
  or ruled out. Post-Brexit the UK is an independent register, so it must be searched separately.
  Source: <https://www.gov.uk/search-for-trademark>

**Register summary:** No exact "Scribed" word-mark was *confirmed* in any of the three registers,
but the searches were **inconclusive** rather than clean, and the surrounding "SCRIB*" landscape
(especially the live, phonetically identical **SCRIBD**) is a real obstacle to registrability and
a source of confusion risk.

## 4. RISK VERDICT — **MEDIUM–HIGH**

| Vector | Level | Reasoning |
|---|---|---|
| **App Store review (name confusion)** | Medium–High | Apple rejects names that are confusingly similar to existing apps/brands. "Scribed" collides with multiple "Scribe" apps on the App Store and with the Scribed AI brand; a metadata/name-similarity rejection is plausible. |
| **Brand / SEO overlap** | High | "Scribed" is dominated in search by **Scribed AI** and the SCRIBD platform, plus a wall of "Scribe-*" transcription tools. A new indie app would be near-invisible and easily mistaken for Scribed AI. |
| **Trademark registrability** | Medium–High | Live phonetically-identical SCRIBD plus a dense SCRIB-/SCRIBE- field in classes 9/42 make a "Scribed" registration vulnerable to refusal or opposition. |
| **Litigation / C&D risk** | Low–Medium | A solo, local-first Mac utility is a small target, but **Scribed AI is in the same vertical using the same name** — a cease-and-desist or App Store complaint is a realistic, low-cost move for them if the app gains visibility. Risk rises with success. |

**Net:** Functional and shippable, but **strategically weak and avoidably risky.** The name is
unprotectable, undiscoverable, and shares the exact niche with an exact-name incumbent.

## 5. RECOMMENDATION — **Rename**

Keeping "Scribed" buys an unownable, hard-to-find name that overlaps an active same-niche
competitor and a major registered brand. For a local-first, bring-your-own-server Mac tool, a
**distinctive, ideally coined** name is worth far more (cleaner trademark path, free domains,
better SEO, no confusion with Scribed AI).

What to aim for: short, pronounceable, **distinctive/coined** (avoid the "Scribe/Scribd/Whisper"
roots entirely), with a free `.app` or `.com` and no App Store twin.

## 6. Before-submission checklist

- [ ] **Pick the final name** from a distinctive/coined shortlist (not the "Scribe*" family).
- [ ] **Formal clearance search** on the chosen name — directly query the live registers
      (not mirrors): USPTO Trademark Search (tmsearch.uspto.gov), UK IPO
      (gov.uk/search-for-trademark), EUIPO eSearch plus, and **TMview** (tmview.org) across the
      EU/UK/US in **Class 9 and Class 42**. Consider a paid clearance/watch service or a trademark
      attorney for a few hundred GBP — cheap insurance vs. a post-launch rebrand.
- [ ] **App Store name check** — search App Store Connect / the live App Store for the exact name
      and close variants; confirm no confusingly-similar app and that the name is available for
      your bundle ID and listing.
- [ ] **Domain + handle sweep** — secure `.app` and/or `.com` plus a fallback (`try<name>.com`),
      and matching GitHub org and social handles, before announcing.
- [ ] **Common-law / web search** — Google the exact name + "app", "transcription", "notes" to
      catch unregistered but active users (common-law rights) that a register search misses.
- [ ] **Decide on filing** — once cleared, consider a **UK IPO** filing (Marc is UK-based; cheapest
      home register) and optionally **USPTO** (intent-to-use) and an **EUTM** if you sell into the
      EU. File in **Class 9** (downloadable software) and **Class 42** (SaaS, if any hosted
      component). File only after clearance confirms the name is open.
- [ ] **Document the clearance** (screenshots/dates) so you have evidence of good-faith adoption.
